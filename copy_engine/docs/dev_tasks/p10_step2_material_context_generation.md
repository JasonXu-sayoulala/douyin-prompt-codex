# P10 Step 2：素材拆解上下文驱动文案生成

## 目标

P10 Step 1 已经完成“爆款素材池到爆点拆解再到转选题”。Step 2 的目标是让素材拆解结果真正进入文案生成，而不是只把素材转成一个普通选题。

本阶段后，系统生成文案时会优先参考：

- 原始素材标题
- 原始素材正文或描述
- 热门评论
- 互动数据
- 爆款潜力分
- 评论争议分
- 冲突点
- 痛点
- 目标人群
- 推荐钩子
- 故事角度
- 改写建议

## 已实现内容

### 1. ContentService 注入素材上下文

`ContentService.build_prompt()` 新增素材上下文注入能力。

当 `Topic.source` 满足 `material:<material_id>:<platform>` 格式时，系统会自动读取 `SourceMaterial` 和最新一条 `MaterialInsight`，并把这些内容追加到生成 Prompt 中。

普通手工选题不受影响。

### 2. 生成页展示素材依据

`contents/generate.html` 新增“素材池依据”卡片。

当选题来自素材池时，页面会展示：素材标题、平台、分类、热度分、争议分、冲突点、痛点、目标人群、推荐钩子，以及返回素材详情入口。

### 3. JSON API 返回来源素材 ID

文案生成 JSON 响应新增 `source_material_id`，方便后续前端或自动化流程追踪内容来源。

## 设计边界

本阶段不做：

- 不新增大模型调用链路
- 不新增独立素材生成接口
- 不改变 `Content` 数据结构
- 不引入新依赖
- 不强制所有选题必须来自素材池

## 验收步骤

1. 进入 `/materials/create` 录入素材。
2. 在素材详情点击“拆解爆点”。
3. 点击“转为选题”。
4. 进入文案生成页。
5. 页面左侧能看到“素材池依据”。
6. 点击生成后，模型 Prompt 会自动带上素材上下文。

## 后续 P10 Step 3 建议

- 在 `Content` 表增加 `source_material_id` 或 `material_insight_id`，实现生成结果可追溯。
- 给素材生成结果加“命中素材点”字段。
- 增加素材 Top 20% 阈值过滤。
- 新增“从素材直接生成文案”按钮，减少转选题后的步骤。
- 增加重复素材检测：按 `source_url` 或标题相似度去重。
