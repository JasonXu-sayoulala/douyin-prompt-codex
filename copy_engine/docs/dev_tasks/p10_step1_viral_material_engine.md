# P10 Step 1：爆款素材池与爆点引擎 MVP

## 理想目标对齐

video-made-proj 的长期目标不是单点“文案生成”或“视频生成”，而是形成一条可持续迭代的内容生产流水线：

> 爆款素材情报 → 爆点拆解 → 选题沉淀 → 文案生成/改写 → 审核 → 导出/媒体生成

P10 Step 1 先补齐“素材情报层”，让系统不再凭空生成内容，而是可以基于对标素材、热门评论和互动数据形成生产输入。

## 本次实现范围

### 新增数据对象

- `SourceMaterial`：爆款素材池
  - 平台、链接、账号、标题、原始文案、热门评论
  - 点赞、评论、分享、收藏
  - `viral_score` 爆款潜力分
  - `discussion_score` 评论争议分
  - 状态：`raw / analyzed / selected / archived`

- `MaterialInsight`：素材爆点拆解结果
  - 冲突点
  - 痛点
  - 目标人群
  - 开头钩子
  - 故事角度
  - 改写建议
  - 风险备注

### 新增服务

- `MaterialInsightService`
  - 本地规则评分，不依赖外部 API
  - 根据文本和互动数据计算热度/争议分
  - 根据关键词推断分类
  - 可将素材一键转为 `Topic`

### 新增页面与路由

- `/materials`：素材列表
- `/materials/create`：录入素材
- `/materials/<id>`：素材详情与爆点拆解
- `/materials/<id>/analyze`：拆解爆点
- `/materials/<id>/to-topic`：转为选题并进入文案生成
- `/materials/<id>/archive`：归档素材

### 首页工作台增强

首页展示：

- 素材总数
- 待拆解素材数
- 已转选题素材数
- 高潜素材 Top 5
- 今日工作流入口

## 边界约束

本阶段不做：

- 不做真实平台爬虫
- 不接抖音/小红书平台 API
- 不改变现有 `ContentService` 生成结构
- 不破坏审核池和媒体生成链路
- 不引入新的三方依赖

## 验收标准

1. 登录后顶部导航能看到“素材池”。
2. 可以手工录入一个素材。
3. 素材详情页可以点击“拆解爆点”。
4. 拆解后能看到冲突点、痛点、目标人群、钩子和评分。
5. 可以点击“转为选题”，进入现有文案生成页。
6. 首页能看到素材池统计和高潜素材。

## 后续 P10 Step 2 建议

- 给 `ContentService.build_prompt` 注入素材拆解上下文。
- 新增“基于素材生成文案”入口。
- 建立素材评分阈值：只允许 Top 20% 进入主生产池。
- 加素材重复检测，避免同一链接重复录入。
- 增加素材榜单视图：按热度、争议、分类、平台聚合。
